Bills & Earn v2 — Full Visual Proposal

📱 4 Purchase Screens

  • Airtime Already built — phone → network → amount → pay
  • Data Needs frontend — phone → network → bundle picker (MTN SME data / Airtel / GLO / 9mobile bundles with prices)
  • Electricity Needs everything — DISCO selector (9 options), meter type (prepaid/postpaid), meter number, amount (₦1k–₦20k)
  • TV Needs everything — Provider (DStv/GOtv/Startimes), bouquet picker with prices, smartcard number

🏗 Backend Work

  • POST /api/v1/bills/electricity — meter number, disco, meter type, amount → call Peyflex → split commission → return tokens
  • POST /api/v1/bills/tv — smartcard, provider, bouquet code → call Peyflex → split commission → return subscription details
  • GET /api/v1/bills/bundles?network=mtn — return available data bundles with prices (from config/env or a DB table)
  • GET /api/v1/bills/discos — return list of DISCOs with meter type support
  • GET /api/v1/bills/tv-bouquets?provider=dstv — return available bouquets with prices
  • Update PeyflexClient — add buy_electricity() and buy_tv() methods
  • Schemas — ElectricityPurchaseRequest, TelevisionPurchaseRequest, BundleResponse, DiscoResponse, BouquetResponse

💵 Revenue Model (no platform money spent)

  • Airtime 3% commission → user 67% → platform 33%
  • Data 3-5% commission → same split
  • Electricity ~1-1.5% commission → same split
  • TV ~1.5-2% commission → same split
  • Total: Platform earns from the aggregator's commission — zero cost to you

🔐 API Key Security

  • Peyflex API key stored in .env as PEYFLEX_API_KEY — never exposed to the client
  • The backend calls Peyflex server-to-server — users never see the key
  • Your key: 013a12a0...3a24b8 (set in .env, used by app/services/peyflex.py)
