Database Schema — New Fields on ContentCatalog

What changes in the existing table

Three new fields are added to the existing content_catalog table. No new tables needed — just columns + a migration.

Field Type Purpose Status
education_level String(50) creche, primary, secondary, tertiary, research NEW
subject String(100) mathematics, physics, english, programming, etc. NEW
license_type String(50) CC BY, CC BY-NC, public_domain, etc. NEW
attribution_text Text "Source: OpenStax, Rice University. CC BY 4.0." NEW
source_url String (unique) Idempotency key — already exists EXISTING
category String(100) Free-text category — already exists EXISTING
parent_work_id FK → content_catalog Links slices to parent book — already exists EXISTING
Why These Fields Matter
  • education_level + subject → lets users filter "Show me Primary 1 Mathematics" or "University Physics"
  • license_type → legal compliance. CC BY allows commercial use with attribution. CC BY-NC does NOT. You must track this per book.
  • attribution_text → displayed in the reader so users see where the book came from (required by CC BY license)

What Attribution Does & How It Works

Legal compliance for open-source content
What Attribution Is

Attribution is the legal requirement of open-source licenses like Creative Commons (CC BY). When you use someone else's openly licensed content, you must credit the original creator. It's like citing a source in an academic paper — but it's a legal requirement, not just good manners.

How Attribution Flows Through Your System

OpenStax API
Returns metadata:
title, author, license
→
Ingestion Module
Saves to DB:
license_type = "CC BY"
attribution_text = "..."
→
ContentCatalog
Row stores
attribution data
→
Reader Screen
Displays:
"Source: OpenStax"
"CC BY 4.0"
License Commercial Use? Attribution Required? Can You Use It?
CC BY 4.0 (OpenStax) ✓ Yes ✓ Yes ✓ YES — with attribution
CC BY-NC (CK-12) ✗ No ✓ Yes ⚠ RISKY — ad revenue = commercial
Public Domain (Gutenberg) ✓ Yes Optional ✓ YES — no restrictions
CC BY-SA (some OER) ✓ Yes ✓ Yes ✓ YES — must share-alike
⚠ Critical Compliance Note

Your app shares ad revenue with users — that's commercial use. CC BY-NC (Non-Commercial) licensed content cannot be used in your app without explicit permission. The license_type field in the database lets your ingestion pipeline automatically reject or flag NC-licensed content before it goes live.

New Database Tables for Social Features

What gets added to the schema
Table Fields Purpose
content_likes id, user_id (FK), content_id (FK), created_at One row per like. Unique(user_id, content_id) prevents double-likes. Same pattern as community_likes.
content_comments id, user_id (FK), content_id (FK), body (Text), parent_comment_id (FK nullable), created_at, updated_at Comments on content slices. parent_comment_id enables replies (threaded comments).
content_comment_likes id, user_id (FK), comment_id (FK), created_at Likes on individual comments. Unique(user_id, comment_id).
content_shares id, user_id (FK), content_id (FK), platform (String), created_at Tracks shares for analytics (which content gets shared most, which platform). Optional but useful for engagement metrics.
New API Endpoints
  • POST /content/{id}/like — toggle like (same pattern as community)
  • GET /content/{id}/comments — paginated comment list
  • POST /content/{id}/comments — post comment (sanitized)
  • POST /content/comments/{id}/like — like a comment
  • POST /content/{id}/share — log share event (analytics)
